KEY DESIGN DECISIONS
================================================================================
1. Data Lake Partitioning by Date
   - Supports incremental loads
   - Enables data lineage tracking
   - Facilitates archival and retention policies

2. Star Schema with Surrogate Keys
   - Dimension tables separated for analytical queries
   - Surrogate keys decouple business keys from system
   - Enables slow-changing dimension handling
   - Optimizes join performance

3. dbt for Transformation
   - Version control for all transformations
   - Automated testing framework
   - Self-documenting models
   - Reproducible data pipeline

4. Async Telegram Scraper
   - Efficient handling of API rate limits
   - Non-blocking I/O for image downloads
   - Detailed error logging for debugging

5. PostgreSQL Raw Layer
   - Preserves original data structure
   - Upsert logic prevents duplicates
   - Audit trail with loaded_at timestamp
   - Foundation for layered architecture
